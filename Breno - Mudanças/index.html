Git e Github. Para que serve e como usar?

Por: Breno Luiz Marques Ferreira

Versionar, modificar e unir trabalhos em grupo, é um dos maiores problemas dos programadores ao redor do mundo!
Tendo em vista que maior parte de nossos trabalhos depende de um grupo, debates e consenso, temos que entender
como as ferramentas GIT podem nos ajudar. De forma simples, o Git se trata de uma base de comandos através de um
terminal, semelhante ao Linux/PowerShell, que utilizamos(maioria das vezes) para clonar e subir arquivos. Clonar,
seria a ação de efetuar o Download do código de um sistema de Repositórios, como o Github ou gitlab... Sendo de
responsabilidade do Administrador do repositório, deixar os códigos mais atualizados o possível. Subir os arquivos
ou dar Push, como dizemos comumente, se refere a capacidade de atualizar os arquivos dentro do Repositório. Temos
como atualizar através dos famosos "PR's" ou Pull Requests, sendo uma espécie de solicitação aos administradores
para alterar os arquivos. O Admin, observaria o que foi alterado e lançaria as atualizações na Master, para que todos
futuramente acessem num novo Clone os arquivos novos.
